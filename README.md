## 博客架构/技术栈
博客目前采用无服务器架构，包含如下几部分：
* **前端：** 基于[Twilight](https://github.com/Spr-Aachen/Twilight)项目深度“魔改”完成构建，在此对原作者表示感谢。采用 Astro + Svelte 混合框架，托管到GitHub仓库并对接Cloudflare Pages实现静态实时构建部署。
* **后端：**
  * 后端采用 Cloudflare D1 数据库作为核心数据库，储存相册信息、照片信息、站点统计、评论区等内容。
  * 通过 Cloudflare Workers 构建起 **相册、评论、站点信息、管理** 四大类API，实现前后端对接与全要素管理。
  * 采用 Cloudflare R2 存储桶作为相册图床，免出站费用，配合D1数据库实现相册高效部署。
* **管理后台：** 基于原生html + CSS + js 三件套，从零手搓博客web管理后台，对接管理API，实现媲美 WordPress 的 CMS 体系。包含完整的**数据看板、相册管理、照片上传、文章管理、文章编辑、评论区管理、友链管理、站点基本配置管理**等功能。但因为用的都是私有API，所以后台代码暂时还没想公开，不在这个仓库里。这个博客后台也有如下亮点，可以分享一下：
  * 通过纯手搓的 TOTP 动态验证机制认证登录，用 Authenticator 输入六位验证码登录后台，还是很酷的！详情可看[这篇文章](https://www.ybjun.com/posts/cf-workers-totp/)。
  * 纯手搓博客主站及后台的相册系统。照片上传模块实现了在前端的照片压缩、转码、加水印并根据CF网络情况“呼吸性”上传的复杂流程。同时相册管理支持照片的各类元数据修改。
  * 经魔改的文章编辑器可快速可视化编辑文章头部YAML元数据，并接受直接复制粘图。粘入的图片通过浏览器缓存转换实现实时显示，并在保存发布文章时自动将文章中所有图链替换为相对链接，然后自动将图片上传到GitHub仓库的文章目录。加载现有文章时亦有替换机制，自动加载文中相对图链对应的图片且不改变原链接。
  * 文章编辑器支持在保存并发布文章后，自动向搜索引擎提交本文章、对应的所有分类和标签页面、博客主页，以及文章归档页面链接的IndexNow请求，方便SEO优化。
  * 纯手搓的简版评论逻辑，采用D1数据库存储评论，同样支持楼中楼、评论审核等多种功能。

📩 **申请邮箱**: `yuebanjun666@outlook.com`

### 📝 申请格式

发送邮件时，请以 **`博客友链添加请求 - [你的站点名称]`** 作为邮件标题，并复制以下模板提供你的站点信息：

```text
网站名称：[填写你的站点名称]
网站描述：[一句话介绍你的网站]
网站链接：[例如 https://www.example.com，请勿以“/”结尾]
头像链接：[提供头像或 Logo 的直链，需确保该链接允许本站 (https://www.ybjun.com) 的CORS跨域请求]
网站标签：[参考下方的标签列表，最多选择 2 个]
```

### 💡 温馨提示
> 1. 如果你提供的是个人联系方式，网站链接处可以使用 `mailto:`协议，如`mailto:123@abc.com`。
> 2. 收到申请后，博主会在近期访问你的站点进行审核。为了保证分类准确，博主可能会根据你的实际网站内容对标签进行调整。
